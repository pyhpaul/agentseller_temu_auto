# 开发环境调试指导

> 面向新工程师：在一台空白机器上把本项目跑起来开发调试。  
> 阅读对象假设：会用 git / 命令行，但没接触过本项目。  
> 预期耗时：5 分钟（不含 BarTender 这种商业软件的下载安装时间）。

---

## 0. 项目是什么

- Chrome MV3 插件（`core/` 公共骨架 + `features/` 多个独立 feature）
- 可选 Python Native Host（仅 `auto_gen_label` 需要，调用 Windows 上的 BarTender 打印标签）
- 当前 4 个 feature：
  - `auto_gen_label`：标签生成 + 合规填写 + 标签主图插入（**唯一需要 native_host + BarTender**）
  - `price_declare`：价格不调整
  - `image_search_1688`：1688 搜图
  - `check_and_publish`：店小秘 ERP 检查与发布

## 5. Chrome 加载插件

1. 打开 `chrome://extensions`
2. 右上角打开**开发者模式**
3. 点**加载已解压的扩展程序**
4. 选择仓库根目录下的 `dist/extension/` 文件夹
5. 加载成功后，扩展卡片会显示一个 **ID**（例如 `dnamfbakkceljnlhiekgjbjchgooaljm`）

> 因为 `core/manifest.template.json` 里有固定的 `key` 字段，无论谁在哪台机器加载 `dist/extension/`，Extension ID 都是同一个固定值。这样 native_host 的 `allowed_origins` 不需要每个开发者改。

## 6. 启动 watch 模式

日常开发跑这一个命令：

```bash
python build/dev.py
```

预期输出：

```
[dev] 启动初始构建...

[watch] monitoring: core/, features/*/content/, features/*/feature.json

```

进程会一直常驻，监听 `core/` 和 `features/*/` 下的源文件变化，自动同步到 `dist/extension/`。

## 7. 改源码 → 看效果（reload 流程）

1. 在编辑器里改 `core/content/registry.js` 或 `features/<feature>/content/index.js`
2. 终端会打印 `[sync] features/xxx/content/index.js → dist/extension/features/xxx/content/index.js`
3. 切到 Chrome，打开 `chrome://extensions`
4. 找到 **AgentSeller for Temu** 卡片，点**右下角的循环箭头 reload**
5. 刷新 Temu 商家中心页面（`https://seller.temu.com/...`），就能看到改动

> `feature.json` 或 `core/manifest.template.json` 变化时，dev.py 会**自动重生 `dist/extension/manifest.json`**，但 Chrome 仍需手动 reload。

### 调试技巧

- **看 content script 日志**：在 Temu 页面按 F12 打开 DevTools，Console 面板。
- **看 service worker 日志**：`chrome://extensions` → 扩展卡片 → 点 **service worker** 链接，会打开独立 DevTools。
- **看源码而非构建产物**：构建脚本会给每个 .js 末尾注入 `//# sourceURL=<源路径>`，DevTools 的 Sources 面板按源码路径展示，等价于 chrome 直接加载源目录。

## 8. （可选）开发 auto_gen_label：注册 dev native host

> 仅当你要联调 `auto_gen_label` 的 BarTender 调用或文件对话框时需要做这一步。  
> 仅在 **Windows** 上执行（WSL 内的开发者：进 Windows 终端跑，或在资源管理器双击 .bat）。

```cmd
cd native_host
dev_install.bat
```

`dev_install.bat` 做的事：

1. 找当前 Python 解释器路径
2. 在 `%APPDATA%\TemuLabel\` 生成一个 `run_host.bat` wrapper，内部调用源码 `main.py`（**不需要先打包 EXE**）
3. 写入 `com.temu.label_host.json` manifest
4. 注册到 `HKCU\Software\Google\Chrome\NativeMessagingHosts\com.temu.label_host`

完成后**重启 Chrome**（native messaging 注册表只在 Chrome 启动时读一次）。

> Native host 日志：`native_host/temu_label_host.log`

### BarTender 装到非默认路径

`bartender_handler.py` 用 `resolve_bt_dll()` 按下面顺序查找 DLL：

1. 环境变量 `TEMU_LABEL_BT_DLL` 指向的文件
2. `C:\Program Files\Seagull\BarTender 2022\Seagull.BarTender.Print.dll`（默认 64-bit）
3. `C:\Program Files (x86)\Seagull\BarTender 2022\Seagull.BarTender.Print.dll`（默认 32-bit）

非默认路径时设置环境变量后重启 Chrome（让 native host 拿到新变量）：

```cmd
setx TEMU_LABEL_BT_DLL "D:\BT2022\Seagull.BarTender.Print.dll"
```

找不到时 `generate_label` 返回 `{success: False, error: '<含已尝试路径的友好提示>'}`，不抛栈。

### Extension ID 不是默认值怎么办

`dev_install.bat` 默认写入 `dnamfbakkceljnlhiekgjbjchgooaljm`。如果你自己 fork 改了 `core/manifest.template.json` 里的 `key` 字段（不推荐），Extension ID 会变。这时手动传参：

```cmd
dev_install.bat <你的实际 extension id>
```

或者修改 `dev_install.bat` 里的 `DEFAULT_EXT_ID`。

## 10. 多人并行开发约束（必读）

**第二个及以后的开发者（或 Claude agent）在本项目工作时，必须建 worktree，不要共用同一工作目录**。

否则一个人的中间状态（如未完成的 `feature.json`）会把其他人的 `build_extension.py` 弄 fail。

详见根 `CLAUDE.md` 的「多 Agent 并发开发硬约束」章节。简要流程：

```bash
git fetch origin --prune
git switch main && git pull --ff-only origin main
git worktree add ../wt-<feature_name> -b feature/<your_branch>
cd ../wt-<feature_name>
# 在 worktree 内跑 dev.py
```

各 worktree 的 `dist/` 互相隔离，watchdog 不占端口可以多个并行。**但 Chrome 一次只能加载一个 worktree 的 dist**，多 worktree 调试时需要在 `chrome://extensions` 里轮流切加载点。

## 常见问题

| 现象 | 原因 / 处理 |
|------|-------------|
| `chrome://extensions` 显示「Manifest file is missing or unreadable」 | 没跑 `build_extension.py`，或加载错了目录（要选 `dist/extension/`，不是仓库根） |
| 改了 `feature.json` 后 Chrome 没生效 | 必须手动 reload 扩展卡片；service worker 也要重启 |
| 改了 content script 但 Temu 页面没反应 | 1) 看终端有没有 `[sync]` 打印；2) reload 扩展；3) 刷新 Temu 页面（content script 只在页面加载时注入） |
| `dev.py` 报错 `ModuleNotFoundError: watchdog` | 先跑 `pip install -r build/requirements-dev.txt` |
| 想调试 `auto_gen_label` 但没 BarTender | 跳过 Phase 1 标签生成相关功能；Phase 2/3 不依赖 native_host，可以单独测 |
| `dev_install.bat` 在 WSL 路径下报错 | 脚本已用 `pushd` 兼容 UNC 路径；如果还有问题，先把代码 clone 到 Windows 本地盘再开发 |
| Chrome 扩展卡片显示「未与本机程序连接」 | 1) 是否跑过 `dev_install.bat` 或 `install.bat`；2) 重启 Chrome；3) 看 `temu_label_host.log` |

## 关键路径速查

| 用途 | 路径 |
|------|------|
| Chrome 加载点 | `dist/extension/` |
| Core 源码 | `core/` |
