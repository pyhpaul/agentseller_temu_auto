# 生产环境使用指导

> 把员工部署包交付到一台空白 Windows 机器上，让员工能正常使用。  
> 本文分两部分：  
> - **A 部分**给**打包人 / 发布工程师**：在 Windows 上出员工部署包。  
> - **B 部分**给**终端员工**：收到部署包后安装到自己机器上。  
> 预期员工端耗时：5 分钟（不含 BarTender 商业软件安装时间）。

---

## A. 打包人：出员工部署包

### 概览：两条打包路径

| 路径 | 触发方式 | 产物位置 | 何时用 |
|------|---------|---------|--------|
| **CI 自动打包**（推荐） | 推 `v*` tag 或在 Actions 页面手动触发 | GitHub Release 资产 / Actions artifact | 正式发布给员工；端到端验证 |
| **本地手动打包**（备用） | Windows 机器跑 `python build\package_all.py` | `dist\TemuLabelSetup.exe` | 没网 / 临时改 / CI 不可用时 |

CI 路径见 **A.7**。本地路径见 A.0～A.6。

### A.0 前置条件

| 项 | 要求 |
|----|------|
| 操作系统 | **Windows 10/11**（必须，PyInstaller 需要在目标平台构建） |
| Python | 3.9+，加入 PATH |
| Git | 任意版本 |
| Inno Setup 6.x（一次性安装） | 用于把部署目录打包成员工双击的 `TemuLabelSetup.exe`。免费下载：https://jrsoftware.org/isdl.php  默认装到 `C:\Program Files (x86)\Inno Setup 6\`，`package_all.py` 会自动检测 |
| BarTender 2022（可选） | 打包不需要，但本机测试 EXE 需要 |

### A.1 装 native_host 的 Python 依赖

```cmd
pip install -r native_host\requirements.txt
```

包含 `pywin32`、`pythonnet`、`Pillow`、`pyinstaller`。

### A.2 装构建脚本依赖

```cmd
pip install -r build\requirements-dev.txt
```

### A.3 拉最新主分支

```cmd
git fetch origin --prune
git switch main
git pull --ff-only origin main
```

### A.4 跑打包脚本

```cmd
python build\package_all.py
```

脚本做四件事：

1. 跑 `build_extension.py` 出 `dist\extension\`
2. 调 `native_host\build\build.bat` 用 PyInstaller 出 `TemuLabelHost.exe`（单文件）
3. 拼装 `dist\TemuLabel_Setup\` 目录
4. **自动把 `dist\TemuLabel_Setup\extension\features\auto_gen_label\content\index.js` 里的 `const TAL_DEBUG = true;` 改为 `false;`**（关闭调试面板）

> 替换有显式校验：找不到 `const TAL_DEBUG = true;` 字面量会**直接报错退出**，避免静默失败。

### A.5 检查产物

```cmd
dir dist
```

应该看到两个交付物：

```
dist\
├── TemuLabelSetup.exe        # **首选交付**：员工双击运行的单文件安装包（约 30-40 MB）
└── TemuLabel_Setup\          # fallback：手动部署目录（员工端无 setup.exe 时用）
    ├── TemuLabelHost.exe
    ├── install.bat
    ├── com.temu.label_host.json
    └── extension\
```

> 没看到 `TemuLabelSetup.exe`？检查是否装了 Inno Setup（参见 A.0），或看 `package_all.py` 末尾是否打印「未找到 Inno Setup，跳过 setup.exe 生成」警告。

### A.6 交付

**首选：单文件交付** — 把 `dist\TemuLabelSetup.exe` 直接发给员工（约 30-40 MB），员工双击即可安装。

**fallback：目录交付** — 把整个 `dist\TemuLabel_Setup\` 打包（zip / 7z）发给员工，员工解压后手动跑 `install.bat`。仅当员工机器无法运行 setup.exe（如杀毒拦截）时使用。

### A.7 CI 自动打包（推荐路径）

GitHub Actions 跑 `windows-latest` runner 自动出 `TemuLabelSetup.exe`。不需要本地装 Python / Inno Setup / pythonnet 等任何东西。

#### A.7.1 触发方式

**正式发布**（产物作为 GitHub Release 资产）：

```bash
git switch main
git pull --ff-only origin main
git tag v1.0.1
git push origin v1.0.1
```

打 tag → GitHub Actions 自动运行 `.github/workflows/build-windows.yml` → 5–10 分钟后 `https://github.com/<repo>/releases` 出新 release，附带 `TemuLabelSetup.exe`。

**手动触发**（仅出 artifact，不创建 release，用于端到端测试）：

GitHub 仓库页 → **Actions** → **Build Windows installer** → **Run workflow** → 选分支跑。
跑完在 workflow run 页面 **Artifacts** 区下载 `TemuLabelSetup-<version>.zip`。

#### A.7.2 Tag 命名规则

**核心约束**：Inno Setup 用 `MyAppVersion` 数字段比较来识别员工机器是否要"升级"。带后缀的版本号退化成字符串比对，员工升级时会被弹窗骚扰或必须手动卸载重装。

| 用途 | 格式 | 例 | Inno 自动升级 |
|------|------|-----|---------|
| **正式发布**（员工部署） | `vMAJOR.MINOR.PATCH` | `v1.0.1` `v1.1.0` `v2.0.0` | ✓ 自动识别 |
| **CI 端到端验证** | `vMAJOR.MINOR.PATCH-rc.N` | `v1.0.1-rc.1` | ✗ 测试包，别给员工 |
| **日常 CI 验证** | 不打 tag，用 workflow_dispatch 手动触发 | — | artifact 下来本地测 |

**三段语义**（按本项目实际划分）：

| 段 | 含义 | 何时升 | 员工感知 |
|----|------|--------|---------|
| **MAJOR** | 员工必须卸载重装才能工作 | native host 协议变 / `com.temu.label_host` 名字变 / 注册表路径变 | 大动作 |
| **MINOR** | 加新 feature 或现有 feature 显著扩展 | 新 feature 目录 / 现有 feature 加大块功能 | 升级后看到新东西 |
| **PATCH** | bug 修复 / UI 微调 / 内部重构 | 单 bug 修复、文案、性能优化 | 静默升级 |

**首发版本**：`installer.iss` 现硬编码 `1.0.0`，已部署机器注册表 DisplayVersion 都是 1.0.0。CI 出的第一个正式包从 **`v1.0.1`** 起步，不要从 `v0.x.y` 开始（会被 Inno 当"降级"）。

**第一次跑 CI**：先推 `v1.0.1-rc.1` 测整条链路（不要直接给员工），通过后再推 `v1.0.1`。

#### A.7.3 版本号自动注入机制

`build/inject_version.py` 解析版本号写入 `deploy/installer.iss` 的 `#define MyAppVersion`：

| 触发场景 | 解析来源 | 例 |
|---------|---------|-----|
| tag push | `GITHUB_REF_NAME` 去 `v` 前缀 | `refs/tags/v1.0.1` → `1.0.1` |
| workflow_dispatch（仓库有 tag） | `git describe --tags --always` | `v1.0.1-3-gabc123` → `1.0.1-3-gabc123` |
| workflow_dispatch（仓库无 tag） | `0.0.0-dev.<short_sha>` | `0.0.0-dev.2842405` |
| 本地直跑（无 tag） | 同上，但仅打 warning 不阻断 | `0.0.0-dev.2842405` |

CI 严格模式（环境变量 `CI=true`）下，dev fallback 会硬退出——防止 tag workflow 误打无版本号的 release 包。

#### A.7.4 CI 失败排查

| 现象 | 排查 |
|------|------|
| `pythonnet` 装失败 | 看 Actions 日志；可能 Python 3.13 不支持新版 pythonnet，临时把 workflow 里 python-version 降到 3.12 |
| Inno Setup 装失败 | `choco install innosetup` 偶发网络问题，re-run workflow 一般就过 |
| `package_all.py` 报「未找到 ISCC.exe」 | 看 chocolatey 是否真把 Inno Setup 装到 `C:\Program Files (x86)\Inno Setup 6\`（默认就是），可能要 `where ISCC` 调试 |
| Release 创建失败 | 看 `softprops/action-gh-release` 日志，权限不够要在仓库 Settings → Actions → Workflow permissions 开「Read and write」 |
| 员工双击 setup.exe 卡在「已安装相同版本」 | 新版 tag 没递增（违反 A.7.2 规则），或员工机器原来就是这个版本 |

## B. 终端员工：在 Windows 机器上安装

> 阅读对象：会用 Windows 桌面、能解压 zip 的员工。  
> 全程预期 5 分钟（不含 BarTender 安装）。

### B.0 前置条件

| 项 | 要求 |
|----|------|
| 操作系统 | Windows 10/11 |
| Chrome 浏览器 | 最新稳定版（[下载](https://www.google.com/chrome/)） |
| BarTender 2022 | **仅使用「标签生成」功能时必装**；其他功能（价格不调整 / 1688 搜图 / 检查与发布）不需要 |

### B.1 双击 `TemuLabelSetup.exe` 安装

1. 下载 / 收到 `TemuLabelSetup.exe`，建议放到 `下载` 或 `桌面`，**双击运行**

> **如果 Windows 弹「Windows 已保护你的电脑 / 未知发行者」蓝色警告**：
> 这不是病毒，是 Windows 对所有"从互联网下载的未签名 .exe"统一弹的 SmartScreen 警告。
> 处理：点蓝框里的 **「更多信息」** → 下方出现 **「仍要运行」** 按钮 → 点击即可继续安装。
> 一次点击，装完后不会再弹。
>
> （根本解决要给安装包做代码签名，属于后续工作，员工先按上面方法绕过。）

2. 按向导操作（全程中文界面）：
   - **欢迎页** → 下一步
   - **安装位置** → 默认 `C:\Users\<你>\AppData\Local\TemuLabel\`，下一步
   - **准备安装** → 点 `安装`
   - 进度条几秒
   - **完成页** → 弹出对话框「安装完成！还需要在 Chrome 完成一步...」**点 `是` 自动打开 chrome://extensions**

> installer 做的事（对员工透明）：
> - 把 `TemuLabelHost.exe` 和扩展文件拷到 `%LOCALAPPDATA%\TemuLabel\`
> - 生成 native messaging host manifest 写到 `%LOCALAPPDATA%\TemuLabel\com.temu.label_host.json`
> - 注册到 Windows 注册表 `HKCU\Software\Google\Chrome\NativeMessagingHosts\com.temu.label_host`

> 安装过程**不需要管理员权限**，企业受限账号也可装。

### B.2 在 Chrome 加载插件

向导自动打开 chrome://extensions 后：

1. 右上角打开**开发者模式**（开关）
2. 点左上角**加载已解压的扩展程序**
3. 选择 `C:\Users\<你>\AppData\Local\TemuLabel\extension\` 文件夹（弹窗里有完整路径提示）
4. 加载后会看到一张「**AgentSeller for Temu**」扩展卡片，ID 通常是 `dnamfbakkceljnlhiekgjbjchgooaljm`

### B.3 关闭并重启 Chrome

native messaging 注册表只在 Chrome **启动**时读一次。完成 B.1 之后必须**完全关闭所有 Chrome 窗口**，再重新打开。

> Windows 任务管理器里确认没有任何 `chrome.exe` 残留进程。

### B.4 （仅用「标签生成」需要）装 BarTender 2022

如果你只用「价格不调整」/「1688 搜图」/「检查与发布」，可以**跳过此步**。

「标签生成」功能依赖 BarTender 2022 的 .NET SDK 调用本地打印模板。安装时优先用默认路径：

- 默认 64-bit：`C:\Program Files\Seagull\BarTender 2022\`
- 默认 32-bit：`C:\Program Files (x86)\Seagull\BarTender 2022\`

安装时确认：

- 安装包含 .NET SDK 组件（`Seagull.BarTender.Print.dll` 必须存在）
- 软件许可证为正版（试用版受限会导致导出文件失败）

**装到非默认路径怎么办**：设置系统环境变量 `TEMU_LABEL_BT_DLL` 指向 DLL 完整路径，重启 Chrome 即可。例：

```cmd
setx TEMU_LABEL_BT_DLL "D:\BT2022\Seagull.BarTender.Print.dll"
```

native_host 优先级：环境变量 → 64-bit 默认 → 32-bit 默认。找不到时「标签生成」会返回友好错误，不会闪退。

### B.5 验证

1. 打开 Temu 商家中心 `https://seller.temu.com/`
2. 登录账号
3. 页面**右下角应该出现一个浮动按钮（FAB）**，**点击展开**是 feature Hub 网格
4. 选择对应功能（例如「标签生成」），按界面提示操作

### B.6 验证 native host 连接（仅「标签生成」需要）

直接在 Hub 里点「标签生成」开始执行最直观：

- 如果 native_host 没连上：会立即弹出错误提示（含「未与本机程序连接」字样），按 B.3 重启 Chrome 重试
- 如果 BarTender DLL 找不到：会提示「未找到 BarTender DLL...」，按 B.4 装 BarTender 或设环境变量

> 进一步排查请联系开发对接人。员工端不需要手动验证 native host 端口。

### B.7 fallback：手动安装（仅 setup.exe 无法运行时）

杀毒软件拦截 / 公司策略禁止运行 .exe / setup.exe 损坏 等场景，可以走传统手动流程：

1. 跟发布人要 `TemuLabel_Setup\` 整个目录（zip / 7z 形式）
2. 解压到固定位置（例：`C:\TemuLabel\TemuLabel_Setup\`），**不要**解压到 `C:\Program Files\` 或 `桌面`
3. 双击 `install.bat`，看到「[成功] Native Host 已注册」即完成
4. 跟 B.2 一样在 chrome://extensions 加载 `TemuLabel_Setup\extension\` 目录
5. 跟 B.3 一样重启 Chrome

> install.bat 把文件装到 `%APPDATA%\TemuLabel\`（注意是 `APPDATA` 不是 `LOCALAPPDATA`，跟 setup.exe 走不同路径），但 chrome 加载点仍指向解压目录的 `extension\` 子目录。

## 常见故障排查（员工端）

| 现象 | 处理步骤 |
|------|----------|
| Chrome 右下角看不到悬浮按钮 | 1) `chrome://extensions` 确认插件**已启用**；2) 刷新 Temu 页面；3) 确认当前页面在 `seller.temu.com` 域名下（其他域名插件不注入） |
| 「未与本机程序连接」/ `native host not connected` | 1) 确认跑过 `install.bat` 且看到「成功」；2) **完全关闭并重启 Chrome**；3) 检查注册表 `HKCU\Software\Google\Chrome\NativeMessagingHosts\com.temu.label_host` 是否存在 |
| `install.bat` 报错 「未找到 TemuLabelHost.exe」 | 你的部署包不完整，找发布人重发完整 `TemuLabel_Setup\` 目录 |
| 标签生成报「未找到 BarTender DLL」 | 1) 装 BarTender 2022（B.4）；2) 装到非默认路径时按 B.4 设置 `TEMU_LABEL_BT_DLL` 环境变量；3) 设完环境变量后**重启 Chrome**让 native_host 重启读到新变量 |
| 标签生成报「BarTender 导出失败」 | 1) 确认 BarTender 2022 正版（试用版限制）；2) 确认 DLL 文件实际存在；3) 看日志 `%APPDATA%\TemuLabel\..` 附近的 `temu_label_host.log` |
| 插件加载后报「Manifest file is missing」 | 加载错了目录，要选 `TemuLabel_Setup\extension\` 这个子目录，不是外层 |
| Chrome 升级后插件失效 | 重新到 `chrome://extensions` 点扩展卡片右下角的循环箭头 reload；如还不行重做 B.3 |
| 装了新版部署包后旧版没自动清掉 | 1) `chrome://extensions` 移除旧扩展卡片；2) 删除旧的 `TemuLabel_Setup\` 目录；3) 重新跑新版 B.1～B.4 |

## 升级流程（已部署机器换新版）

**setup.exe 流程**（首选）：

1. 发布人按 A 部分出新的 `TemuLabelSetup.exe`
2. 员工**先关闭 Chrome**
3. 员工双击新版 `TemuLabelSetup.exe`（Inno Setup 会自动检测旧版并覆盖安装到同一目录）
4. 按向导走完
5. 重启 Chrome，`chrome://extensions` 找到扩展卡片点 reload

**手动流程**（fallback）：跟初次安装一样走 B.7。

> 因为 `manifest.template.json` 中固定了 `key` 字段，Extension ID 不会因为重装而变化，已注册的 native host `allowed_origins` 仍然有效。

## 卸载

**setup.exe 安装的**（首选）：

1. **控制面板** → **程序和功能** → 找「Temu Label Helper」→ **卸载**
2. 卸载程序会自动清理 `%LOCALAPPDATA%\TemuLabel\` 目录和注册表项
3. `chrome://extensions` 找到「AgentSeller for Temu」卡片，点**移除**

**手动 install.bat 安装的**：

1. `chrome://extensions` 找到扩展卡片，点**移除**
2. 删除 `%APPDATA%\TemuLabel\` 整个目录
3. 删除注册表项：
   ```cmd
   reg delete "HKCU\Software\Google\Chrome\NativeMessagingHosts\com.temu.label_host" /f
   ```
4. 删除部署包解压目录（例如 `C:\TemuLabel\TemuLabel_Setup\`）

BarTender 2022 是独立软件，按其他用途决定是否一并卸载。

## 关键路径速查（员工端）

**setup.exe 安装的**：

| 用途 | 路径 |
|------|------|
| 安装目录 | `%LOCALAPPDATA%\TemuLabel\`（= `C:\Users\<你>\AppData\Local\TemuLabel\`） |
| Chrome 加载点 | `%LOCALAPPDATA%\TemuLabel\extension\` |
| Native host EXE | `%LOCALAPPDATA%\TemuLabel\TemuLabelHost.exe` |
| Native messaging manifest | `%LOCALAPPDATA%\TemuLabel\com.temu.label_host.json` |
| 注册表项 | `HKCU\Software\Google\Chrome\NativeMessagingHosts\com.temu.label_host` |
| BarTender DLL 自定义路径 | 环境变量 `TEMU_LABEL_BT_DLL` |
| BarTender DLL 默认 64-bit | `C:\Program Files\Seagull\BarTender 2022\Seagull.BarTender.Print.dll` |
| BarTender DLL 默认 32-bit | `C:\Program Files (x86)\Seagull\BarTender 2022\Seagull.BarTender.Print.dll` |

**手动 install.bat 安装的**：

| 用途 | 路径 |
|------|------|
| 部署包解压目录 | `C:\TemuLabel\TemuLabel_Setup\`（或员工解压位置） |
| Chrome 加载点 | `C:\TemuLabel\TemuLabel_Setup\extension\` |
| Native host EXE | `%APPDATA%\TemuLabel\TemuLabelHost.exe`（注意 `APPDATA` 不是 `LOCALAPPDATA`） |
| Native messaging manifest | `%APPDATA%\TemuLabel\com.temu.label_host.json` |
