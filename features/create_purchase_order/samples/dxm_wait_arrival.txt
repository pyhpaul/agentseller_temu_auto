### 搜索按钮定位策略
⚠️ 页面有「高级搜索」展开区，里面也有「搜索」按钮。
**必须限定在 kwInput 所在的 `.search-container-main` / `.searchContainer` 容器内**
取 `button[type="submit"]`，不能全局 `document.querySelector('button[type="submit"]')`。

```js
const scope = kwInput.closest('.search-container-main, .searchContainer') || document;
const searchBtn = scope.querySelector('button[type="submit"]')
  || U.findByText('button, .ant-btn', '搜索', scope);
```

## 关键设计决策：found 判断方式

⚠️ **表格行文本不直接包含 SKU 货号**（货号要点「货品详情」才展示）。
因此 found **不能**用 `行.textContent.includes(skuNo)` 判断，
而是用**「有数据行 + 无可见暂无数据空态」**组合判断：

```js
const rows = document.querySelectorAll('.vxe-body--row');
const emptyShown = Array.from(document.querySelectorAll('.vxe-table--empty-block, .empty-container'))
  .some(e => e.getBoundingClientRect().height > 0 && /暂无数据/.test(e.textContent));
if (rows.length > 0 && !emptyShown) { found = true; break; }
```

用 `getBoundingClientRect().height > 0` 判断空态可见（而非只看 DOM 存在），
避免"空态节点存在但不显示"的假阳性。

## 非阻断语义

搜不到商品行只发 warn toast + `return { ok: true, found: false }`，
**不 return `{ ok: false }`**，因为采购单号已在审核弹窗阶段取得（Phase 2 核心目标已达成）。
用户可手动在待到货页查找。
