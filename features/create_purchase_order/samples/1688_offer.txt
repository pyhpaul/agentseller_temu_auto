1688 商品详情页 (https://detail.1688.com/offer/<id>.html) 标题取值基线

===== 实测样本 =====
- og:title = undefined            ← 该页【没有】 og:title meta，不能作主源
- h1 = 义乌市谨硕日用品有限公司     ← 是【店铺/公司名】，不是商品标题，【不可用】
- document.title = 家用冰箱保鲜鸡蛋盒带盖抽屉式厨房储物盒可叠加滚蛋式鸡蛋收纳盒 - 阿里巴巴
                                   ← 商品标题 + " - 阿里巴巴" 后缀  ← 唯一可靠源

===== 取值策略（CPO_READ_1688_TITLE） =====
1. 风控页早退：path 含 /punish 或 search 含 x5secdata
2. 主源：document.title 去掉 /\s*[-_|]\s*(阿里巴巴|1688).*$/ 后缀 → 全标题
   （循环等待 ~4s，避开加载中占位 "阿里巴巴"）
3. 退路：og:title（个别页面有）
4. h1 明确【不用】（是店铺名）

===== 决策：取全标题，不缩短 =====
用户确认取完整商品标题填入店小秘「中文名称」；嫌长在「保存前核对」环节手动删短。
（曾考虑末尾截取/品类词启发式提取核心品名，但关键词堆砌标题无可靠确定性规则，放弃自动缩短）

===== 教训 =====
原 plan 假设 og:title 优先、h1 兜底——实测 og:title 空会 fallback 到 h1=店铺名 → 取错。
印证项目铁律：改 selector / 取值前必须 dump 真实数据，不能信"页面应该长这样"的直觉。
