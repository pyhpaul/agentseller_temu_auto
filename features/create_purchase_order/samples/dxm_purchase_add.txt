## ── 已存在弹窗（已实机抓取）──
当输入已入库的1688订单号并点获取时出现：
<div class="ant-modal css-1oz1bg8 dxm-modal" role="document" style="width:650px">
  <div class="ant-modal-content">
    <div class="ant-modal-header">
      <div class="ant-modal-title">提示</div>
    </div>
    <button class="ant-modal-close"><span class="ant-modal-close-x">...</span></button>
    <div class="ant-modal-body">
      <div id="commonProgressBarStatus" class="f-red">已完成!</div>
      <div id="commonProgressBarDetail">3304702093890742986采购单号：PO1SLPT027932已存在，不能重复添加</div>
    </div>
    <div class="ant-modal-footer">
      <button class="ant-btn ant-btn-default"><span>关闭</span></button>
    </div>
  </div>
</div>

结论：Task 5 关键词检测 /已存在|不能重复添加|已完成/ 命中「已完成!」+「已存在，不能重复添加」；
关闭按钮文本「关闭」可达（footer 内 .ant-btn-default，或角上 .ant-modal-close X）。
当前实现用鲁棒关键词匹配（/已存在|不能重复添加|已完成/）+ 可见性检测（height>0）。
注意：检测容器选择器去掉了 .ant-modal-confirm-body，避免命中内层子元素导致关闭按钮 querySelector 落空。

补充：弹窗正文「采购单号：PO...」措辞 ≠ 旧的「采购单：PO...」，
cpo-logic.js extractPoNo 正则已加「号?」兼容两种（见 fix commit）。

## ── 轮询逻辑说明 ──
CPO_P2_ADD_FETCH 点击「获取1688订单」后，轮询最多 25 次（每次 200ms，共约 5s）：
- 检测到已存在弹窗 → 点关闭 → 返回 { ok: true, exists: true }（bg 走已存在分支）
- 超时未出现弹窗   → 返回 { ok: true, exists: false }（bg 靠跳转监听接管后续 edit 页）
