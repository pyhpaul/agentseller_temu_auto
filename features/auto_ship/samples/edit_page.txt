===== EDIT_PAGE（装箱发货 inner 编辑页，真实 dump 2026-05-28）=====

⚠️ 第三种容器：Drawer（[data-testid="beast-core-drawer"] / [data-testid="beast-core-drawer-content"]），
   右侧抽屉，非 modal、非 popover。标题「装箱发货」。
   —— topModal/topPopover 都定位不到，要用 topDrawer。

字段（form 内，每个 form-item 有稳定 id，比 hash class 可靠）：

| 字段 | form-item id | 控件 | 操作 |
|------|-------------|------|------|
| 发货方式 | deliverMethod | radioGroup（自送/自行委托第三方/在线物流下单） | 已默认「在线物流下单」(data-checked=true)，不动 |
| 预估总重量* | predictTotalPackageWeight | inputNumber value=1 | 已预填，不动 |
| 预估总体积 | predictPackageVolume | inputNumber | 选填，不动 |
| **包装方式** | **packagingType** | **radioGroup（箱子/袋子/箱子和袋子）** | **选「箱子和袋子」** |
| **发货总箱/包数*** | **expressPackageNum** | **inputNumber（placeholder 填写交接给快递公司的预估箱子数）** | **填 1** |
| 发货仓库* | deliveryAddressId | select value=KEY086 | 已预填，不动 |
| 物流公司* | expressCompanyId | select（disabled） | 不动 |
| 预约取货时间* | expectPickUpGoodsDate/Time | datePicker/timePicker | **填箱数后日期自动补、时间需手选 18:00**（见文末） |

包装方式 radio 结构：
<div id="packagingType" data-testid="beast-core-form-item">
  ...<label class="Form_itemLabelContent">包装方式</label>...

    <label data-testid="beast-core-radio" data-checked="false">...<div class="RD_textWrapper_5-111-0 RD_prevRadio_5-111-0">箱子</div></label>

===== 预约取货时间 timePicker（真实 dump 2026-05-29，补充）=====

⚠️ 旧 dump 写「disabled 不动」是**填箱数之前**的状态。实测：填完「发货总箱/包数」后，
   Temu 自动把日期补进 #expectPickUpGoodsDate（input value="2026-05-29"），
   但时间 #expectPickUpGoodsTime 仍为空，必须手选（业务要求，默认 18:00）。

时间输入框（drawer 内，#expectPickUpGoodsTime）：
  <div data-testid="beast-core-timePicker-input" data-status="active">
    <input mode="hh:mm" placeholder="请选择时间" readonly type="text"
           data-testid="beast-core-timePicker-html-input" value="">     ← readonly！不能 setInputValue
    <svg data-testid="beast-core-icon-time-circle">（suffix 时钟图标，点它也能弹下拉）
  </div>

点开后下拉（⚠️ 挂 document 级 portal，不在 drawer 内）：
  <div data-testid="beast-core-portal" class="...PT_dropdown... PP_dropdown..." style="left:..;top:..">
    <ul data-testid="beast-core-timePicker-list-hh">
      <li class="cIL_item cIL_medium cIL_active cIL_disabled">00</li>   ← active+disabled（不可选）
      <li class="cIL_item cIL_medium cIL_disabled">01</li> ... 13（全 disabled，已过/业务限制）
      <li class="cIL_item cIL_medium">14</li> ... <li>18</li> ... <li>23</li>   ← 14-23 可选，选 18
    </ul>
    <ul data-testid="beast-core-timePicker-list-mm">
      <li class="cIL_item cIL_medium cIL_active cIL_disabled">00</li>
      <li class="...cIL_disabled">01</li> ... 59   ← ⚠️ 选「时」前分钟列表全 disabled！
    </ul>                                              选完 18(时)后 mm 列表才异步刷新出可选 00
  </div>

无「确定」按钮 → 选完时+分即回填 input.value（受控，DOM 可读，可能 "18:00" 或 "18:00:00"）。
交互：openTimeDropdown(点 input/suffix/容器多重兜底) → clickTimeListItem('hh','18')
      → 轮询等 mm 刷新 → clickTimeListItem('mm','00') → 写后读校验 input.value startsWith '18:00'。
disabled 判定：li.className 含 'disabled'（cIL_disabled）即排除；cIL_active 是当前高亮态，不代表可选。
